# Contributing to pyclef

Thank you for considering contributing to `pyclef`! Contributions are what make open-source projects thrive, and we appreciate your interest in improving this project. Whether it's reporting a bug, suggesting a feature, or submitting a pull request, your contributions are welcome.

---

## How to Contribute

### 1. Report Issues
If you encounter a bug, have a question, or want to suggest a new feature:
- Check the [issue tracker](https://github.com/paul-shuvo/pyclef/issues) to see if the issue has already been reported.
- If not, create a new issue with the following details:
  - A clear and descriptive title.
  - Steps to reproduce the issue (if applicable).
  - Expected behavior and actual behavior.
  - Any relevant code snippets, logs, or screenshots.

### 2. Suggest Features
We welcome feature suggestions! If you have an idea for a new feature:
- Open a new issue in the [issue tracker](https://github.com/paul-shuvo/pyclef/issues).
- Provide a detailed description of the feature and its use case.
- If possible, include examples of how the feature would be used.

### 3. Submit Code Changes
If you'd like to contribute code to the project:
1. **Fork the Repository**:  
   Click the "Fork" button on the top-right corner of the repository page to create your own copy of the repository.

2. **Clone Your Fork**:  
   Clone your forked repository to your local machine:
   ```bash
   git clone https://github.com/paul-shuvo/pyclef.git
   cd pyclef
